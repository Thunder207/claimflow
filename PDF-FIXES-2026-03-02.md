## Bugs Fixed

### 1. Garbled Emoji Characters in Receipt Titles
- **Symptom:** Flight receipt title showed `'├╛ Flight`, hotel showed `├α<├θ├ρ Hotel #1`
- **Cause:** PDFKit's default Helvetica font cannot render Unicode emojis (έεΙΎ╕Π, ΏθΠρ, ΏθγΗ, etc.)
- **Fix:** Replaced emojis with plain text labels in `modeLabels` and `getReceiptLabel()`
- **Result:** Titles now read "ATTACHED RECEIPT έΑΦ Flight", "ATTACHED RECEIPT έΑΦ Hotel #1", etc.

### 2. Duplicate Hotel Receipts in PDF
- **Symptom:** Hotel receipt images appeared twice έΑΦ once from `transport_receipts` table (hotel_0 mode) and again from `expenses.receipt_data` BLOB
- **Cause:** PDF generation fetched hotel receipts from both sources independently
- **Fix:** Added `e.expense_type !== 'hotel'` filter to `expensesWithReceipts` to skip hotel expenses (already covered by transport_receipts)

### 3. Blank Duplicate Pages (10 ghost pages)
- **Symptom:** PDF had 20 pages instead of 10 έΑΦ pages 11-20 were blank with only footer text
- **Cause:** PDFKit `bufferPages: true` + `switchToPage()` footer loop created ghost duplicate pages (known PDFKit bug)
- **Fix:** 
  - Removed `bufferPages: true` from PDFDocument options
  - Removed `switchToPage()` footer loop entirely
  - Footers now added via **pdf-lib** post-generation: load finished PDF έΗΤ `drawText()` on each page έΗΤ save
  - This also handles footer addition for merged PDF receipt pages

## Architecture Change

### Before (broken)
```
PDFKit (bufferPages:true) έΗΤ switchToPage() footer loop έΗΤ doc.end() έΗΤ pdf-lib merge PDF receipts
```

### After (fixed)
```
PDFKit (no buffering) έΗΤ doc.end() έΗΤ pdf-lib: merge PDF receipts + add footers to ALL pages έΗΤ save
```

**Key insight:** All post-processing (receipt merging + footers) now happens in a single pdf-lib pass after PDFKit is done. This is cleaner and eliminates the bufferPages duplication bug entirely.
